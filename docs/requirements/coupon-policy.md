# 🎟️ Coupon 도메인 요구사항

선착순 쿠폰 발급, 쿠폰 사용 및 정책 제한 관련 요구사항을 정의합니다.

---

## ✅ 기능 목록

### 1. 쿠폰 발급 API

- 사용자는 선착순으로 할인 쿠폰을 발급받을 수 있다.
- 발급 가능한 총 수량이 존재하며, 소진되면 더 이상 발급되지 않는다.
- 한 사용자는 같은 쿠폰을 1회만 발급 가능하다.

### 2. 보유 쿠폰 조회 API

- 현재 사용자가 보유한 쿠폰 목록을 조회할 수 있다.
- 쿠폰 정보에는 ID, 할인율/금액, 사용 여부, 만료일 등이 포함된다.

### 3. 주문 시 쿠폰 사용

- 쿠폰을 제출하면 해당 주문 금액에 할인 적용이 가능하다.
- 사용된 쿠폰은 **재사용 불가** 처리되어야 한다.

---

## ⚠️ 정책 및 제약사항

### 🎯 쿠폰 발급

- 발급 가능한 수량이 0 이하일 경우 `409 Conflict`
- 중복 발급 시도 시 `409 Conflict`
- 동시 발급 요청이 들어올 수 있으므로, **선점 락 또는 atomic 연산 등**으로 경쟁 조건 방지 필요

### 💸 쿠폰 사용

- 사용된 쿠폰은 재사용 불가 (`409 Conflict`)
- 유효하지 않은 쿠폰 ID 입력 시 `404 Not Found`
- 사용 시점에 유효하지 않은 쿠폰이면 `422 Unprocessable Entity`
    - 예: 만료일 지남, 해당 사용자의 쿠폰이 아님

| 구분    | 시나리오            | 예상 결과                    |
