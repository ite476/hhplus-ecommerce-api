# 🛒 Order 도메인 요구사항

상품 주문, 결제 처리 및 외부 데이터 전송 관련 요구사항을 정의합니다.

---

## ✅ 기능 목록
### 1. 주문 생성 API
- 사용자 식별자, (상품 ID, 수량) 목록을 입력받아 주문을 수행한다.
- 결제는 기 충전된 잔액을 기준으로 수행하며,
  - 성공 시: 잔액 차감 + 재고 차감 + 주문 기록 저장
  - 실패 시: 아무 처리도 되지 않아야 한다 (롤백)
- 결제 후 주문 내역을 외부 데이터 플랫폼으로 전송해야 한다.

> 주문, 결제가 동일 트랜잭션에서 이루어지기 때문에  

### 2. 주문 목록 조회 API

- 사용자 식별자를 입력받아 전체 주문 목록을 조회한다.
  - `주문 ID`
  - `주문 결제 상태`

---

## ⚠️ 정책 및 제약사항

### 💰 잔액 정책
- 결제 총액 > 사용자 잔액 → `409 Conflict`
- 결제 성공 시, 반드시 잔액 차감되어야 함
### 📦 재고 정책
- 수량 부족한 상품이 하나라도 존재하면 전체 주문 실패
- 성공 시 상품별 재고 감소
- 주문 시 상품 개수 입력 범위 제한
  - 최소: `1`
  - 최대: `100`
### 💳 결제 원자성
- 주문, 결제, 재고 차감은 동일 트랜잭션 내에서 처리
- 예외 발생 시 전체 롤백 보장
### 📊 데이터 플랫폼 전송
- 결제 성공 후, 외부 시스템으로 주문 내역 전송
- 외부 시스템은 실제로 존재하지 않으므로 **Mock** 또는 **Fake Module** 사용 가능
- 비동기 처리 여부는 자유

## 📘 API Spec

### 1. 주문 및 결제 생성 API
> - 엔드포인트: `POST /order`
> - 입력:
>   - `header`
>     - 사용자 식별자 `필수`
>   - `body`
>     - 상품 배열
>       - 상품 코드 `필수`
>       - 구매 개수 `필수`
>     - 할인 쿠폰 코드 `선택`
> - 출력:
>   - `성공` `201 Created`
>     - `header`
>       - 리소스 위치 `GET /order/{orderId}`
>   - `실패`
>     - 존재하지 않는 사용자에 대한 주문 요청 `404 Not Found`
>     - 존재하지 않는 상품에 대한 주문 요청 `422 Unprocessable Entity`
>     - 구매하려는 상품의 재고가 부족함 `422 Unprocessable Entity`
>     - 구매 개수 값이 올바르지 않음 `422 Unprocessable Entity`
>       - 최소 허용: `0L`
>       - 최대 허용: `100L`
>     - 할인 쿠폰 관련:
>       - 참조: [할인 쿠폰 요구사항](coupon-policy.md#-정책-및-제약사항)
>       - 유효하지 않은 쿠폰 ID 입력 시 `404 Not Found` 
>       - 사용된 쿠폰을 입력함 `422 Unprocessable Entity`
>       - 사용 시점에 유효하지 않은 쿠폰이면 `422 Unprocessable Entity`
>     - 결제 관련:
>       - 사용자 포인트 잔고가 부족함 `422 Unprocessable Entity`

### 2. 주문 목록 조회 API
> - 엔드포인트: `GET /order`
> - 입력:
>   - `header`
>     - 사용자 식별자 `필수`
>   - `body`
>     - 주문 배열
>       - 상품 코드 `필수`
>       - 구매 개수 `필수`
> - 출력:
>   - `성공` `200 OK`
>     - `body`
>       - 주문
>         - 주문 코드
>         - 주문 결제 완료 상태
>   - `실패`
>     - 존재하지 않는 사용자에 대한 조회 요청 `404 Not Found`

## 🧪 테스트 시나리오 예시

| 구분    | 시나리오            | 예상 결과                            |
|-------|-----------------|----------------------------------|
| ✅ 정상  | 주문 가능한 상품 2개 주문 | 잔액 차감 + 재고 차감 + 주문 성공            |
| ⚠️ 실패 | 상품 중 1개 재고 부족   | 전체 주문 실패 (`409`)                 |
| ⚠️ 실패 | 잔액 부족           | 주문 실패 (`409`)                    |
| ⚠️ 예외 | 데이터 전송 실패       | 주문은 성공, 전송 실패는 로깅/재시도 등 보완 처리 가능 |
