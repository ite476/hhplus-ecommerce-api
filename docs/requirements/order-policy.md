# 🛒 Order 도메인 요구사항

상품 주문, 결제 처리 및 외부 데이터 전송 관련 요구사항을 정의합니다.

---

## ✅ 기능 목록

### 1. 주문 & 결제 API

- 사용자 식별자, (상품 ID, 수량) 목록을 입력받아 주문을 수행한다.
- 결제는 기 충전된 잔액을 기준으로 수행하며,
    - 성공 시: 잔액 차감 + 재고 차감 + 주문 기록 저장
    - 실패 시: 아무 처리도 되지 않아야 한다 (롤백)
- 결제 후 주문 내역을 외부 데이터 플랫폼으로 전송해야 한다.

---

## 📦 입력 예시

```json
{
  "userId": 1,
  "items": [
    { "productId": 3, "quantity": 2 },
    { "productId": 4, "quantity": 1 }
  ],
  "couponId": 10 // (선택)
}
```

---

## ⚠️ 정책 및 제약사항
### 💰 잔액 정책
- 결제 총액 > 사용자 잔액 → `409 Conflict`
- 결제 성공 시, 반드시 잔액 차감되어야 함

### 📦 재고 정책
- 수량 부족한 상품이 하나라도 존재하면 전체 주문 실패
- 성공 시 상품별 재고 감소
### 💳 결제 원자성
- 주문, 결제, 재고 차감은 동일 트랜잭션 내에서 처리
- 예외 발생 시 전체 롤백 보장
### 📊 데이터 플랫폼 전송
- 결제 성공 후, 외부 시스템으로 주문 내역 전송
- 외부 시스템은 실제로 존재하지 않으므로 **Mock** 또는 **Fake Module** 사용 가능
- 비동기 처리 여부는 자유

## 🧪 테스트 시나리오 예시

| 구분    | 시나리오            | 예상 결과                            |
|-------|-----------------|----------------------------------|
| ✅ 정상  | 주문 가능한 상품 2개 주문 | 잔액 차감 + 재고 차감 + 주문 성공            |
| ⚠️ 실패 | 상품 중 1개 재고 부족   | 전체 주문 실패 (`409`)                 |
| ⚠️ 실패 | 잔액 부족           | 주문 실패 (`409`)                    |
| ⚠️ 예외 | 데이터 전송 실패       | 주문은 성공, 전송 실패는 로깅/재시도 등 보완 처리 가능 |
