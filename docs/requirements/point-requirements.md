# 🪙 Point 도메인 요구사항

사용자의 **포인트(잔액)** 충전 및 조회 기능과 관련된 요구사항을 정의합니다.

---

## ✅ 기능 목록

### 1. 잔액 충전 API

- 사용자 식별자와 금액을 입력받아 해당 사용자의 잔액을 충전한다.
- 기존 잔액에 금액을 더하는 방식으로 처리한다.
- 충전 금액은 양수만 허용한다.

### 2. 잔액 조회 API

- 사용자 식별자를 기준으로 현재 잔액을 조회한다.
- 응답은 사용자 ID, 현재 잔액을 포함한다.

---

## ⚠️ 정책 및 제약사항

- 포인트 충전 금액 범위 제한
  - 최소: `0`
  - 최대: `1,000,000`

## 📘 API Spec

### 1. 잔액 충전 API
> - 엔드포인트: `PATCH /point/charge`
> - 입력:
>     - `header`
>       - 사용자 식별자 `필수`
>     - `body` 
>       - 충전 금액 `필수`
> - 출력:
>     - `성공` `204 No Content`       
>       - `header` 
>         - 리소스 위치 `GET /point`         
>     - `실패`
>         - 존재하지 않는 사용자에 대한 충전 요청 `404 Not Found`
>         - 충전 금액 범위가 올바르지 않은 경우 `422 Unprocessable Entity`

### 2. 잔액 조회 API
> - 엔드포인트: `GET /point`
> - 입력:
>   - `header`
>     - 사용자 식별자 `필수`
>   - 출력:
>     - `성공` `200 OK`
>       - `body`
>         - 사용자 식별자
>         - 포인트 잔액
>     - `실패`
>       - 존재하지 않는 사용자에 대한 조회 요청 `404 Not Found`

| 구분    | 시나리오                  | 예상 결과                    |
|-------|-----------------------|:-------------------------|
