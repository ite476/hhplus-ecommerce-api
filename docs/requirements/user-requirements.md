# 👤 User 도메인 요구사항

사용자의 **포인트(잔액)** 충전 및 조회 기능과 관련된 요구사항을 정의합니다.

---

## ✅ 기능 목록

### 1. 잔액 충전 API

- 사용자 식별자와 금액을 입력받아 해당 사용자의 잔액을 충전한다.
- 기존 잔액에 금액을 더하는 방식으로 처리한다.
- 충전 금액은 양수만 허용한다.

### 2. 잔액 조회 API

- 사용자 식별자를 기준으로 현재 잔액을 조회한다.
- 응답은 사용자 ID, 현재 잔액을 포함한다.

---

## ⚠️ 정책 및 제약사항

- **존재하지 않는 사용자**에 대한 충전/조회 요청 시 `404 Not Found` 반환
- **음수 금액**으로 충전 요청 시 `400 Bad Request` 반환
- 충전/조회 대상 사용자는 내부적으로 자동 생성되거나, 사전 등록되어 있다고 가정 가능

---

## 🧪 테스트 시나리오 예시

| 구분    | 시나리오                  | 예상 결과           |
|-------|-----------------------|-----------------|
| ✅ 정상  | 1번 사용자에게 10,000원 충전   | 잔액: 10,000      |
| ✅ 정상  | 동일 사용자에게 추가 5,000원 충전 | 잔액: 15,000      |
| ⚠️ 예외 | -1,000원 충전 시도         | 400 Bad Request |
| ⚠️ 예외 | 존재하지 않는 사용자 조회        | 404 Not Found   |

```json
}
```

조회 응답 예시

```json
{
```
